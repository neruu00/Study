# SW 문제 해결

## 문제 해결 과정

- 1. 문제를 읽고 이해한다.
  - 속도 : 살짝 문제 유형?
  - 2회독 : 정독 - 제약사항, 입력의 크기, 시간 복잡도, 공간 복잡도
  - 3회독 : 정독 - 놓친 제약사항, 꼼꼼한 설계를 동반한
- 2. 문제를 익순한 용어로 재정의한다.
- 3.  어떻게 해결할지 계획을 세운다.
- 4. 계획을 검증한다.
  - 체크할 수 있는 테스트케이스 검증
  - 엣지케이스 검증
- 5. 프로그램으로 구현한다.
- 6. 어떻게 풀었는지 돌아보고, 개선할 방법이 있는지 찾아본다.

## 무엇이 좋은 알고리즘인가
- 정확성 : 얼마나 정확하게 동작하는가
- 작업량 : 얼마나 적은 연산으로 원하는 결과를 얻어내는가
- 메모리 사용량 : 얼마나 적은 메모리를 사용하는가
- 단순성 : 얼마나 단순한가
- 최적성 : 더 이상 개선할 여지없이 최적화되었는가

## 시간 복잡도
연산의 작업량, 수행 시간
- 최선의 경우 
  - 빅 오메가 표기법 사용 
  - 최선의 시나리오로 최소 이만한 시간이 걸림
- 최악의 경우
  - 빅 오 표기법 사용
  - 최안의 시나링로 아무리 오래 걸려도 이 시간보다 덜 걸림
- 평균적인 경우
  - 빅 세타 표기법 사용
  - 평균 시간을 나타냄

## java.util.Scanner
데이터를 읽어 구분자로 토큰화하고 다양한 타입으로 변환해 리턴함

대량의 데이터를 처리 시 수행 시간이 비효율적임

### 주요 메서드

| 메서드명     | 특징                                                                                            |
| ------------ | ----------------------------------------------------------------------------------------------- |
| nextInt()    | int 타입 반환. <br> 유효 문자열 후 white space를 만나면 유효 문자열 까지 반환                   |
| nextDouble() | double 타입 반환. <br> 유효 문자열 후 white space를 만나면 유효 문자열 까지 반환                |
| next()       | 문자열 반환. 유효 문자열 후 white space를 만나면 유효 문자열 까지 반환                          |
| nextLine()   | 문자열 반환. 유효 문자열 후 개행 문자 만나면 개행 문자까지 처리. next와 달리 띄워쓰기 포함 가능 |

## java.lang.StringBuilder
- 문자열의 조작을 지원하는 클래스
- 자바에서 상수로 취급되는 문자열을 조작 시마다 새로운 문자열이 생성되는 것을 방지해줌
- append()
- toString()
- Thread Safe 없음
  - 동기화 x
  - 알고리즘을 풀때는 멀티 스레드 환경 아니니 상관없음

```java
StringBuilder sb = new StringBuilder();
sb.append("Hello");
sb.append("SSAFY").append("!!");
System.out.println(sb.toString());

sb.setLength(sb.length()-2);
System.out.println(sb.toString());
```
