## 자바 기본

### 변수 Variable

자료를 저장하기 위한 메모리 공간

자바의 변수는 데이터 형태 즉 Type으로 구분

- 타입에 따라 메모리 공간의 크기가 달라짐
- 데이터 타입에 따라 크게 2가지로 분류

| 기본형 | 참조형 |
| --- | --- |
| 미리 정해진 크기의 데이터 표현 | 크기가 미리 정해질 수 없는 데이터의 표현 |
| 변수에 값 자체 저장 | 변수에 실제 값을 참조할 수 있는 주소만 저장 |

자바에서는 포인터를 알 수 없다

자바에서 한 비트는 sign bit (양/음 구분)

| 구분 | Type | bit 수 | 값 |
| --- | --- | --- | --- |
| 논리형 | boolean | - | true / false |
| 정수형 | int | 32 | -2^31 ~ 2^21-1 |
| 실수형 | float | 32 | float f=0.123...789F; |
| 실수형 | double | 64 | double d=0.123...789; |
| 문자형 | char | 16 | 2^16 |

### 실수의 문제점

2진수로 저장 시 값 손실 발생. 정문도 문제

- 유효 자리수를 이용한 반올림
- 정수로 변환
- BigDecimal 객체 사용

### 형변환

기본형은 기본형끼리, 참조형은 참조형 끼리 가능

#### 명시적 형변환

자료에 대한 손실 발생

```java
double d = 100.5;
int result = (int)d
```

#### 묵시적 형변환

자료에 대한 손실 발생 x

```java
byte b = 10;
int i = (int)b;
int i2 = b;
```

형변환에서 타입의 표현 범위가 커지는 경우 묵시적 형변환 가능

값과 타입의 크기는 신경쓰지 않는다.

```java
byte b1 = 10;
byte b2 = 20;
byte b3 = b1 + b2;
//Type mismatch: cannot convert from int to byte

byte b3 = (byte)(b1 + b2); //cast to byte
int b3 = b1 + b2 // change b3 type to int
```

```java
int i1 = 10;
int l1 = 20;
int i2 = i1 + l1;
//Type mismatch: cannot convert from long to int

int i2 = (int)(i1 + l1) // cast to int
long i2 = i1 + l1 // change i2 type to long
```

```java
float f1 = 10.0;
float f2 = f1 + 20.0;
//Type mismatch: cannot convert from double to float

float f1 = 10.0f; //change value
float f1 = (float)10.0; //cast to float
double f1 = 10.0; //change f1 type to double

float f2 = f1 + 20.0f; // change value
float f2 = (float)(f1 + 20.0); //cast to float
double f2 = f1 + 20.0; //change f2 type to double
```

- 산술이항 연산자는 연산 전에 피 연산자의 타입을 일치시킨다.
- 피연산자의 크기가 4byte(int) 미만이면 int로 변경한 후 연산 진행
- 두 개의 피연산자 중 큰 타입으로 형 변환 후 연산 진행

### switch

```java
int month = 3;
int day = -1;
switch (month) {
    case 2:
        day = 29;
        break;
    case 4:
    case 6:
    case 9:
    case 11:
        day = 30;
    default:
        day = 31;
}

// switch expressions
int month = 3;
int day = switch (month) {
    case 2 -> 29;
    case 4, 6, 9, 11 -> 30;
    default -> {
        yield 31;
    }
}
```
