# 추상 클래스 Abstract Class

상속을 전제로 설계된 클래스

추상 메서드는 있을 수도 있고 없을 수도 있음

## 특징

추상 클래스는 직접 객체 생성이 불가함. 대신 하위 클래스에 의해 간접적으로 객체를 생성할 수 있음
- 상태를 가질 수 있음
- 부분 구현에 적합
  - 공통 로직 + 일부 강제(추상 메서드)
- 하위 클래스 간 공통된 실행 흐름을 제공(템플릿 메서드)

```java
abstract class Animal{
    private final String name;

    public Animal(String name) {
        this.name = name;
    }

    //공통 기능(이미 구현됨)
    public void intro() {
        System.out.println("나는 "+name);
    }

    //추상 메서드
    //재정의 강제
    public abstract String sound();

    //템플릿 메서드
    //전체 흐름을 고정. 일부 메서드(sound)만 하위 클래스에 위임
    public final void perform() {
        intro();
        System.out.println(sound());
    }
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }

    @Override
    public String sound() {
        return "bark";
    }
}

class Cat extends Animal {
    public Cat(String name) {
        super(name);
    }

    @Override
    public String sound() {
        return "meow";
    }
}
```

## 추상 클래스틑 사용하는 이유

abstract 클래스는 `구현의 강제`를 통해 프로그램의 안정성 향상

abstract 클래스를 상속 받는 경우

- abstract 메서드를 강제로 구현
- abstract 클래스로 변경

# 인터페이스 interface

객체가 어떤 역할로 사용될 수 있는지 정의한 것

- 모든 멤버 변수는 public static final이며 생략 가능
- 모든 메서드는 public abstract이며 생략 가능

## 장점
```java
class SamsungPay implements Payment, Serlializable {...}
```

- 다중 구현
  - 여러개의 타입을 가질 수 있다(다형성)
- 구현체가 아니라 타입(interface)에 의존
  - 구현제를 쉽게 교체 가능
    - 확장성 향상
  - 코드간의 결합도가 낮아짐
    - 코드가 수정되었을 때 다른 코드에 영향이 적음(느슨한 결합)


## 인터페이스 상속

클래스와 마찬가지로 `extends`를 이용해서 상속 가능

클래스와 차이점은 다중 상속 가능

# 추상 클래스 VS 인터페이스
 - 공통 변수/기능 + 일부 강제 -> 추상 클래스
 - 객체의 역할 명세 + 다중 구현 + 교체 가능성 -> 인터페이스
