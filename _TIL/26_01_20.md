# 추상화 Abstraction

## 객체 지향 Object Oriented Programming

### 객체지향 언어의 특징

| 특징 | 설명 |
| --- | --- |
| 추상화 | 현실의 객체를 추상화해서 클래스를 구성함 |
| 다형성 | 하나의 객체를 여러 가지 타입으로 참조함 |
| 상속 | 부모 클래스의 자산을 물려받아 자식을 정의함으로 코드의 재사용이 가능 |
| 데이터 은닉과 보호 | 데이터를 외부에 직접 노출시키지 않고 메서드를 이용해 보호 |

### 객체지향 프로그램밍이란?

객체란?

- 주체가 아닌 것, 주체가 활용하는 것

객체지향 프로그래밍

- 주변에 많은 것들을 객체화 해서 프로그래밍 하는 것
- 객체를 많이 만드는 것을 추천

변경과 재사용이 간단함

### 객체지향 프로그래밍의 장점

블록 형태의 모듈화된 프로그래밍

- 신뢰성 높은 프로그래밍
- 추가/수정/삭제 용이
- 재사용성 높음

### JVM의 메모리 정보

#### meta space

클래스 정보 처리

- 타입 정보
- 필드 정보
- 메서드 정보
- 상수 풀

#### stack

메서드들의 실행 공간

- 스레드 별로 별도 관리
- 메서드 호출 시 마다 메서드 프레임 적층
- 메서드 프레임에 로커 변수도 쌓이는 구조

#### heap

객체를 저장하기 위한 영역

- 스레드에 의해 공유
- 객체가 생성되고 GC에 의해 정리 됨

객체는 heap에 생성되고 공유됨
